# 核心概念

Ordine 的实体模型围绕可组合性设计。小型、良好类型化的组件组合成强大的自动化工作流。

## 实体层次

```
最佳实践 ──► 操作 ──► 流水线 ──► 规则
    │          │         │
    └── 清单    └── 技能   └── 任务 (执行)
```

## 最佳实践

**最佳实践** 捕获编码规范，附带检查清单。清单中的每一项可以通过脚本检查或 LLM 驱动的分析来验证。

- 定义为结构化数据（标题、描述、清单项）
- 可在多个操作中复用
- 与代码库一起版本控制

## 操作

**操作** 是一个原子级编码任务，具有可配置的执行器后端。

```
操作 = 执行器配置 + 输入 Schema + 输出 Schema
```

执行器类型：
- **`agent`** — AI agent（Claude 或 Codex），可配置系统提示词和工具
- **`script`** — 自定义脚本执行
- **`rule-check`** — 策略验证

Agent 模式：
- **`skill`** — 使用注册的技能进行结构化 AI 执行
- **`prompt`** — 基于直接提示词的 AI 执行

## 流水线

**流水线** 是由类型化节点通过边连接的有向无环图（DAG）。

### 节点类型

| 类型 | 说明 |
|------|------|
| `folder` | 目录输入 — 读取文件夹树 |
| `code-file` | 单文件输入 |
| `operation` | 执行一个操作 |
| `output-local-path` | 将输出写入本地目录 |
| `compound` | 组合多个节点 |
| `condition` | 条件分支 |
| `github-project` | GitHub 仓库输入 |

### 执行模型

1. 节点被组织成 **执行层级**（拓扑排序）
2. 同一层级的节点 **并行** 执行
3. 数据沿边从父节点流向子节点
4. 每个节点产生 `NodeCtx`（content + inputPath）供下游消费

## 技能

**技能** 是可插拔的 AI agent 能力。技能定义：

- 唯一 ID 和标签
- 技能功能描述
- 分类类别

技能被操作引用，驱动 agent 执行。

## 规则

**规则** 是可以自动触发流水线执行的策略。规则将事件（如代码变更）连接到流水线执行。

## 任务

**任务** 跟踪流水线运行的执行。包括：

- 状态跟踪（queued → running → done/failed）
- 实时追踪和日志
- 结构化输出（JSON 结果）
- 持续时间和时间元数据
